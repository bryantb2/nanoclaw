# Plan 03.1-03: Linear OAuth App + Bot Persona

## Objective
Switch the Linear integration from personal API key to OAuth app token, and add bot persona
instructions to groups/global/CLAUDE.md so Fleet knows how to behave as a Linear bot.

## Context
- `container/agent-runner/src/index.ts` uses `LINEAR_API_KEY` env var with Bearer auth to `https://mcp.linear.app/mcp`
- The personal API key (`lin_api_...`) in `data/env/env` works but makes Fleet act as a personal user
- An OAuth app token makes Fleet appear as a named bot application in Linear comments/updates
- `groups/global/CLAUDE.md` and `groups/CLAUDE.md` must stay in sync

## Tasks

### Task 1: Add Linear bot persona instructions (auto)
Add a "Bot Persona in Linear" subsection to the Linear Integration section in `groups/global/CLAUDE.md`.
Instructs Fleet to write comments in third-person/impersonal style and clarifies it acts as an OAuth app.

Done criteria:
- `groups/global/CLAUDE.md` has "Bot Persona in Linear" subsection
- Instructions cover: impersonal comment style, no user impersonation, framing questions as bot

### Task 2: Sync groups/CLAUDE.md (auto)
Apply the same edit to `groups/CLAUDE.md` to keep it in sync with `groups/global/CLAUDE.md`.

Done criteria:
- `groups/CLAUDE.md` is identical to `groups/global/CLAUDE.md`

### Task 3: Create Linear OAuth app (checkpoint:human-action)
User must create a Linear OAuth application in the browser to get an OAuth token.

Steps:
1. Go to https://linear.app/settings/api/applications/new
2. Create application named "Fleet Bot" (or similar)
3. Set redirect URI to `http://localhost:3000/callback` (placeholder — OAuth flow not needed, just token)
4. For NanoClaw, use the "Actor Token" approach: Settings > API > OAuth Applications > your app > Actor tokens
5. Generate an actor token (this acts on behalf of the workspace, appears as the app)
6. Copy the token

### Task 4: Update LINEAR_API_KEY on server (auto, post-checkpoint)
Replace `LINEAR_API_KEY` in `~/nanoclaw/data/env/env` on the production server with the new OAuth token.

Command:
```bash
ssh agentfleet@100.104.163.53 "sed -i 's/^LINEAR_API_KEY=.*/LINEAR_API_KEY=<NEW_TOKEN>/' ~/nanoclaw/data/env/env"
```
Then restart nanoclaw: `systemctl --user restart nanoclaw`

Done criteria:
- `data/env/env` on server has updated LINEAR_API_KEY
- NanoClaw restarted with new token

## Success Criteria
- [ ] groups/global/CLAUDE.md has bot persona instructions
- [ ] groups/CLAUDE.md is in sync
- [ ] LINEAR_API_KEY on server uses OAuth token (not personal API key)
- [ ] NanoClaw restarted and running
