# Plan 02-01: KICKOFF Execution

## Objective

Set up NanoClaw on the production server and execute all 11 KICKOFF steps to deploy the Fleet PM agent with full infrastructure configuration.

## Context

NanoClaw is a lightweight AI agent system. The KICKOFF.md defines 11 sequential steps to configure it for production use with Slack, containers, Linear/GitHub integrations, and operational features.

## Tasks

### Task 1: Initial Setup
- Run `/setup` to install NanoClaw on production server
- Run `/add-slack` to register Slack channels: main DM, dev-team, fleet-ops
- Configure sender allowlist for U0982HUBQ56

### Task 2: KICKOFF Steps 2-11
Execute all remaining KICKOFF steps:
1. Container memory limits (3 GB per container)
2. Queue acknowledgment (queued message when channel busy)
3. GitHub CLI in container Dockerfile
4. Linear MCP server in container agent runner
5. Credential pass-through (ANTHROPIC_API_KEY, LINEAR_API_KEY, GITHUB_TOKEN via JWT)
6. Git identity in containers
7. uploadFile IPC action
8. Fleet-ops group creation
9. Nightly review cron (weekdays 6 PM Pacific)
10. in_flight_tasks SQLite table + interrupted task notification

### Task 3: Finalize and Commit
- Restart NanoClaw service
- Verify service is running
- Commit and push all changes to bryantb2/nanoclaw fork

- NanoClaw running on production server
